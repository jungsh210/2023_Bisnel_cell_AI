# Expected Results

이 문서는 실행 후 어떤 결과를 봐야 하는지 설명합니다.

## 1. 이 프로젝트 결과는 어디에 나오나요?

현재 코드 기준으로 결과는 주로 아래 두 곳에서 확인합니다.

- 콘솔(터미널, 명령 프롬프트)
- 마지막에 뜨는 그래프 창

즉, 지금 구조는 결과 파일 자동 저장보다 “실행 중 출력 확인”이 중심입니다.

## 2. 콘솔에서 보는 주요 결과

학습 중에는 이런 값들이 보입니다.

- `train Loss`
- `train Acc`
- `val Loss`
- `val Acc`

마지막에는 이런 값이 보입니다.

- `test Loss`
- `test Acc`

## 3. 각 값의 의미

### train Loss

훈련 데이터에서 모델이 얼마나 틀리는지 보여줍니다.  
보통 학습이 진행될수록 점점 작아지는 것이 좋습니다.

### train Acc

훈련 데이터에서 정답을 얼마나 맞추는지 보여줍니다.  
보통 학습이 진행될수록 점점 커지는 것이 좋습니다.

### val Loss

검증 데이터에서의 오차입니다.  
훈련용 데이터가 아닌 데이터에서도 잘 작동하는지 보는 지표입니다.

### val Acc

검증 데이터에서의 정확도입니다.  
실제로 중요한 값 중 하나입니다.

### test Loss / test Acc

최종 평가 결과입니다.  
실험을 마무리할 때 가장 중요하게 참고할 수 있는 값입니다.

## 4. 좋은 결과란 무엇인가요?

보통 아래와 같은 흐름이면 좋은 편입니다.

- train Loss 감소
- val Loss도 너무 크게 증가하지 않음
- train Acc 증가
- val Acc도 함께 증가하거나 안정적으로 유지
- test Acc가 충분히 높음

## 5. 이상한 결과 예시

### 경우 1. train은 좋은데 val/test가 낮음

가능한 해석:

- 과적합 가능성
- 데이터가 적거나 편향되었을 가능성

### 경우 2. train도 val도 둘 다 낮음

가능한 해석:

- 모델이 충분히 학습되지 못함
- 데이터 구조나 라벨 품질 문제
- 설정값이 적절하지 않을 가능성

### 경우 3. loss가 이상하게 출렁이거나 커짐

가능한 해석:

- 학습률 또는 데이터 문제
- 입력 데이터 품질 문제
- 배치 크기 영향 가능성

## 6. 마지막 그래프는 어떻게 읽나요?

코드는 마지막에 2개의 그래프를 보여줍니다.

### 왼쪽 그래프

- Val Accuracy
- Train Accuracy

이 그래프는 epoch가 증가할수록 정확도가 어떻게 변하는지 보여줍니다.

### 오른쪽 그래프

- Val Loss
- Train Loss

이 그래프는 epoch가 증가할수록 오차가 어떻게 변하는지 보여줍니다.

## 7. 결과를 보고 다음에 무엇을 하나요?

결과를 보면 다음 방향을 생각할 수 있습니다.

- 모델 종류 변경
- epoch 수 변경
- batch size 변경
- 데이터셋 재정리
- 데이터 증강 방식 조정
- 전처리 방식 검토

즉, 결과는 “끝”이 아니라 다음 실험의 출발점입니다.

## 8. 현재 코드의 한계

현재 구조에서는 아래가 부족합니다.

- 결과 파일 자동 저장
- 모델 체크포인트 자동 저장
- 실험 설정 기록 자동 저장
- 로그 정리 자동화

따라서 처음 실행하는 사람은 실행 후 아래를 직접 기록하는 것이 좋습니다.

- 사용한 데이터 경로
- 사용한 모델명
- epoch 수
- batch size
- 최종 test Acc
- 그래프 스크린샷

이 점은 후속 연구를 위해 반드시 더 보완될 필요가 있습니다.
