# Project Big Picture

## 1. 프로젝트 목표

이 프로젝트의 목표는 뉴런 이미지 데이터를 분류하는 딥러닝 모델을 학습하고, 그 모델이 데이터를 얼마나 잘 구분하는지 확인하는 것입니다.

여기서 중요한 것은 단순히 코드를 실행하는 것이 아니라, 다음 질문에 답하는 것입니다.

- 어떤 데이터를 분류하려고 하는가?
- 어떤 모델을 사용하고 있는가?
- 학습이 잘 되었는가?
- 최종 결과를 보고 다음 실험을 어떻게 바꿔야 하는가?

## 2. 입력과 출력

### 입력

입력은 이미지 데이터입니다.  
코드는 `train`, `val`, `test` 폴더 안에 클래스별 이미지가 정리되어 있다고 가정합니다.

예를 들어 개념적으로는 이런 구조입니다.

```text
데이터 루트/
  train/
    class0/
    class1/
  val/
    class0/
    class1/
  test/
    class0/
    class1/
```

### 출력

이 프로젝트의 출력은 현재 코드 기준으로 아래와 같습니다.

- 학습 중 콘솔에 출력되는 epoch별 `Loss`, `Acc`
- 마지막 테스트 결과 `test Loss`, `test Acc`
- 학습/검증 정확도 및 손실 그래프

즉, “파일로 저장된 리포트”보다 “실행 중 보이는 숫자와 그래프”가 핵심 결과입니다.

## 3. 전체 흐름

이 저장소의 큰 흐름은 아래와 같습니다.

1. `Experiments_Neuron.py`에서 실험 설정을 정합니다.
2. `LoadData.py`가 이미지 데이터를 읽어서 DataLoader를 만듭니다.
3. `TrainModels.py`가 모델을 초기화하고 학습/검증을 수행합니다.
4. `TestModel.py`가 최종 테스트 데이터를 이용해 성능을 확인합니다.
5. 마지막에 그래프로 학습 과정을 시각화합니다.

## 4. train, val, test는 왜 나누나요?

- `train`: 모델이 실제로 배우는 데이터
- `val`: 학습 중간에 성능을 확인하는 데이터
- `test`: 마지막에 최종 평가만 하는 데이터

이렇게 나누는 이유는 모델이 외워버린 것인지, 실제로 잘 분류하는 것인지 구분하기 위해서입니다.

## 5. Accuracy와 Loss는 무엇인가요?

### Accuracy

정답을 맞춘 비율입니다.  
예를 들어 100개 중 90개를 맞추면 정확도는 0.90 또는 90%입니다.

### Loss

모델이 얼마나 틀렸는지를 수치로 나타낸 값입니다.  
일반적으로 Loss는 낮을수록 좋습니다.

보통 기대하는 패턴은 다음과 같습니다.

- train Loss: 점점 감소
- val Loss: 너무 심하게 증가하지 않음
- train Accuracy: 점점 증가
- val Accuracy: 함께 증가하거나 어느 수준에서 안정화

## 6. 이 결과를 보고 무엇을 하려는 건가요?

실행 후 얻은 결과를 바탕으로 다음 판단을 하려는 것입니다.

- 현재 모델이 이 문제에 적절한가?
- 데이터 양이나 품질이 충분한가?
- 과적합이 있는가?
- 다른 모델이나 다른 전처리를 시도해야 하는가?

즉, 이 저장소는 단순히 “돌아가는 코드”가 아니라, 분류 성능을 비교하고 개선 방향을 잡기 위한 실험 도구라고 이해하면 됩니다.

## 7. 현재 코드의 특징

현재 코드에는 아래 특징이 있습니다.

- 설정값이 `Experiments_Neuron.py`에 직접 들어 있습니다.
- 데이터 경로가 하드코딩되어 있습니다.
- 결과 저장 자동화가 강하지 않습니다.
- 초보자가 바로 이해하기 쉽도록 정리된 구조는 아직 부족합니다.

그래서 이번 문서화의 목적은 처음 받은 사람도 이 구조를 이해하고, 최소한 “무엇을 하고 있는 코드인지”를 놓치지 않게 만드는 것입니다.
