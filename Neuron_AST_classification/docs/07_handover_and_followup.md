# Handover And Follow-up

이 문서는 채영이 작업을 포함한 현재 프로젝트 상태를 후속 연구 관점에서 정리한 문서입니다.

## 1. 왜 이 문서가 필요한가?

현재 저장소는 코드가 완전히 없는 상태는 아니지만, 후속 연구자가 바로 받아서 안정적으로 이어가기에는 정리 부족한 부분이 있습니다.

특히 처음 받은 사람 입장에서는 아래가 문제입니다.

- 어떤 파일부터 봐야 하는지 불분명
- 데이터 경로가 하드코딩되어 있음
- 실행 결과를 어디에 기록해야 하는지 명확하지 않음
- 실험 설정 관리가 체계화되어 있지 않음

즉, 단순히 “코드가 있다”는 것과 “후속 연구가 가능하다”는 것은 다릅니다.

## 2. 현재 상태에서 좋은 점

- 메인 실험 흐름이 한 파일에서 보이기 때문에 전체 흐름을 한 번에 보기 쉽습니다.
- 데이터 로딩, 학습, 테스트가 분리되어 있어 구조를 따라가기 시작할 수는 있습니다.
- 기본적인 실험은 실행 가능한 형태로 남아 있습니다.

## 3. 현재 상태에서 부족한 점

### 1) 데이터 경로 하드코딩

`Experiments_Neuron.py`에 작성자 환경 기준 경로가 직접 들어 있습니다.

문제:

- 다른 컴퓨터에서 바로 실행 불가
- 문서가 없으면 초보자는 어디를 바꿔야 하는지 모름

### 2) 실험 설정 관리 부족

현재 실험 설정은 코드 안에 직접 적혀 있습니다.

문제:

- 어떤 조건으로 돌렸는지 기록하기 어려움
- 반복 실험 비교가 어려움

### 3) 결과 저장 체계 부족

현재 결과는 주로 콘솔 출력과 그래프로 확인합니다.

문제:

- 나중에 비교하기 어려움
- 실험 기록 누락 가능성 큼

### 4) 초보자 친화성 부족

Python/PyTorch를 잘 모르는 학생은 코드의 목적, 실행 순서, 결과 의미를 동시에 이해하기 어렵습니다.

## 4. 후속 연구 전에 먼저 정리하면 좋은 것

우선순위 기준으로 정리하면 아래와 같습니다.

### 우선순위 1. 문서화

이번 작업처럼 실행 방법, 결과 해석, 파일 역할을 정리해야 합니다.

### 우선순위 2. 경로 분리

데이터 경로를 코드 밖에서 설정 가능하게 바꾸면 재사용성이 훨씬 좋아집니다.

### 우선순위 3. 결과 저장

아래를 자동 저장하는 기능이 있으면 좋습니다.

- 실험 설정
- 최종 accuracy/loss
- 그래프 이미지
- 모델 weight

### 우선순위 4. 실험 관리 체계화

예:

- config 파일 사용
- 로그 폴더 구조 정리
- 실험 이름 규칙 정리

## 5. 교수님 관점에서 왜 이 부분이 중요한가?

교수님 말씀처럼, 이 부분이 잘 정리되어 있어야 다음 학생이 설명을 반복해서 듣지 않고도 프로젝트를 이어갈 수 있습니다.

즉, 좋은 인수인계는 단순히 파일 전달이 아니라 아래를 가능하게 해야 합니다.

- 학생이 스스로 실행 가능
- 학생이 결과를 읽을 수 있음
- 학생이 다음 실험을 스스로 설계할 수 있음

## 6. 이번 문서화의 역할

이번 문서화는 다음 문제를 줄이는 역할을 합니다.

- “어디서부터 봐야 하지?”
- “무슨 결과가 나오면 정상이지?”
- “이 실험이 왜 필요한 거지?”
- “어떤 파일이 중요한 거지?”

다만 이것만으로 모든 문제가 해결되는 것은 아닙니다.  
후속 연구를 더 안정적으로 이어가려면 구조 개선도 eventually 필요합니다.

## 7. 후속 연구자를 위한 한 줄 조언

이 저장소는 먼저 “무슨 실험을 하려는 코드인지”를 이해하고, 그 다음에 “실행”, 그 다음에 “개선” 순서로 접근하는 것이 가장 안전합니다.
